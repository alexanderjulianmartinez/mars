# Salience Memory v2 — Proposal (not implemented)

Salience Memory v1 (and its expanded benchmark) measures **retrieval quality** of
salience-weighted vs similarity-only ranking over a static, hand-labeled corpus
with mocked execution. This document proposes directions for v2. **It is a
proposal only** — nothing here is implemented, and several items depend on
capabilities owned by Cortex/AutoDev, not Mars.

Guiding constraint: Mars stays the **measurement layer**. Anything that generates
context or executes tasks lives behind the Cortex/AutoDev provider boundary; v2
adds *experiments and metrics*, not new retrieval algorithms or runtimes inside Mars.

## 1. Larger and harder corpora

- Scale to 100+ queries / 2,000+ memories across more domains, generated by the
  same deterministic authoring tool.
- **Measured** distractor difficulty: after embedding, keep only distractors whose
  cosine similarity to the target exceeds a threshold, so "plausible" is verified
  in embedding space rather than asserted by the author.
- Graded relevance labels (target=2, support=1, other=0) to use the full power of
  nDCG instead of binary relevance.
- Cross-query interference: a single shared memory pool across queries (not
  per-query candidate sets), so retrieval must discriminate globally.

## 2. Temporal decay

- Add explicit timestamps to memories and a **time-aware** retrieval arm whose
  recency weight decays continuously, vs a static-recency baseline.
- Benchmark whether decay helps the agent prefer the *latest* decision when an
  older, equally-similar memory contradicts it (the stale-vs-current case the v1
  corpus already seeds).
- Metric: "freshness@k" — fraction of retrieved relevant memories that are the
  current version rather than a superseded one.

## 3. Contradiction resolution

- The expanded corpus already includes a `contradictory` category. v2 would add a
  retrieval arm that detects contradiction pairs and surfaces the **resolved**
  memory, measuring how often the contradicted/stale claim is suppressed.
- This aligns with Cortex's existing `find_contradictions` / `resolve` surface, so
  the experiment would *consume* that capability rather than reimplement it.
- Metric: "contradiction-avoidance@k" — rate at which a known-false contradictory
  memory is kept out of the top-k.

## 4. Memory confidence decay

- Use the `confidence` signal (already on every memory) in ranking and study its
  decay over time/accesses: low-confidence memories should fall unless reinforced.
- Arm comparison: confidence-aware ranking vs salience-only, on a corpus seeded
  with `low_confidence` memories that are occasionally correct.
- Metric: precision of high-confidence retrievals vs realized correctness.

## 5. Long-horizon agent execution (beyond retrieval-only)

- v1 mocks execution. v2 would connect the retrieval arm to **real AutoDev runs**
  on multi-step tasks, measuring whether better memory retrieval produces better
  *task outcomes* (acceptance criteria, diff quality) — the Track A scorers — not
  just better retrieval metrics.
- This closes the loop: retrieval quality → context quality → execution quality.
- Cost and flakiness make this expensive; it should be a small, seeded,
  reproducible suite gated behind an explicit opt-in, never a default CI run.

## 6. Statistical rigor

- Run each arm over N seeded trials and report paired bootstrap CIs + effect size
  (reuse Apollo's `compare_arms`) so a recall delta comes with a confidence
  interval, not a point estimate.
- A power analysis to justify the query count (the v1 expanded corpus uses 30
  queries without one).

## Sequencing

1. Confirm the expanded v1 benchmark discriminates on a real embedded run.
2. Add graded relevance + measured distractor difficulty (cheap, Mars-only).
3. Temporal decay and contradiction-avoidance arms (depend on Cortex signals).
4. Long-horizon execution coupling (depends on AutoDev; most expensive, last).

Everything above is deferred. The immediate, in-scope next step remains: run the
**existing** expanded corpus against real Cortex semantic retrieval and verify the
discrimination estimate in `docs/SALIENCE_MEMORY_V1_EXPANDED.md`.
